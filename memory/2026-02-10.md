### 飞书插件修复（00:17-00:26 UTC+8）
**问题：** 更新后飞书连不上

**诊断过程：**
1. 检查openclaw status，发现duplicate plugin警告：
   ```
   Config warnings:
   - plugins.entries.feishu: plugin feishu: duplicate plugin id detected
   ```
2. 发现两个feishu插件：
   - 系统自带：`/root/.nvm/versions/node/v22.22.0/lib/node_modules/openclaw/extensions/feishu/`
   - 用户自定义：`~/.openclaw/extensions/feishu/`
3. 两个插件ID都是"feishu"，导致冲突

**解决方案：**
- 删除用户自定义插件：`rm -rf ~/.openclaw/extensions/feishu/`
- 尝试重启Gateway（但gateway工具被禁用）

**执行结果：**
- ✅ duplicate plugin警告消失
- ✅ 系统自带feishu插件完好
- ✅ 配置文件中feishu.enabled: true
- ❌ Channels列表仍为空

**后续建议：**
需要手动重启OpenClaw服务才能完全加载插件：
```bash
pkill -f openclaw
openclaw
```

**经验总结：**
- 插件ID冲突会导致通道无法加载
- 删除冲突插件后需要Gateway完全重启
- `openclaw status`可以诊断插件冲突
