## 工作流程

### 设计流程
1. **需求分析** - 理解产品需求和用户场景
2. **竞品分析** - 研究同类产品设计模式
3. **信息架构** - 设计页面结构和导航
4. **原型设计** - 创建交互原型
5. **视觉设计** - 完成高保真设计稿
6. **设计评审** - 与产品、开发团队评审
7. **设计交付** - 提供设计规范和资源
8. **可用性测试** - 验证设计效果

### 协作流程
- 与产品经理确认需求
- 与前端开发确认技术可行性
- 与测试团队制定可用性测试方案
- 收集用户反馈并迭代设计

## 设计原则

### 移动端设计原则
1. **简洁性** - 界面简洁，重点突出
2. **一致性** - 保持设计语言统一
3. **反馈性** - 操作有明确反馈
4. **效率性** - 减少用户操作步骤
5. **容错性** - 允许用户犯错并恢复

### 股票应用设计要点
1. **数据可视化** - 股票数据清晰展示
2. **实时性** - 强调实时数据更新
3. **对比性** - 涨跌数据对比明显
4. **可操作性** - 交易相关操作便捷
5. **信息密度** - 合理的信息展示密度

```css
/* 股票应用颜色系统 */
:root {
  /* 品牌色 */
  --primary-color: #1677ff;
  --primary-hover: #4096ff;
  
  /* 状态色 */
  --success-color: #52c41a;  /* 上涨 */
  --error-color: #ff4d4f;    /* 下跌 */
  --warning-color: #faad14;
  
  /* 中性色 */
  --text-primary: #262626;
  --text-secondary: #8c8c8c;
  --border-color: #d9d9d9;
  --background-color: #f5f5f5;
  
  /* 数据可视化 */
  --up-gradient: linear-gradient(135deg, #52c41a, #73d13d);
  --down-gradient: linear-gradient(135deg, #ff4d4f, #ff7875);
}
```

## 交互设计模式

### 手势交互
1. **下拉刷新** - 数据更新
2. **上拉加载** - 分页加载
3. **左滑删除** - 列表项操作
4. **长按操作** - 更多选项
5. **双击点赞** - 快速操作

### 动效设计
1. **页面过渡** - 平滑的页面切换
2. **加载动画** - 数据加载反馈
3. **状态变化** - 按钮状态变化
4. **数据更新** - 实时数据变化
5. **操作反馈** - 用户操作确认

## 可用性测试

### 测试方法
1. **启发式评估** - 尼尔森十大可用性原则
2. **认知走查** - 模拟用户完成任务
3. **A/B测试** - 不同设计方案对比
4. **眼动追踪** - 用户注意力分析
5. **用户访谈** - 收集定性反馈

### 测试指标
- 任务完成率
- 任务完成时间
- 错误率
- 用户满意度
- 系统可用性量表(SUS)

## 最佳实践

### 移动端设计最佳实践
1. **拇指友好** - 重要操作在拇指可及范围
2. **减少输入** - 尽量使用选择而非输入
3. **即时反馈** - 所有操作都有即时反馈
4. **渐进式披露** - 复杂功能逐步展示
5. **错误预防** - 防止用户犯错的设计

### 股票应用设计最佳实践
1. **数据准确性** - 确保数据准确无误
2. **实时更新** - 股票数据实时更新
3. **对比明显** - 涨跌数据对比清晰
4. **操作便捷** - 交易相关操作简单
5. **风险提示** - 必要的风险提示
